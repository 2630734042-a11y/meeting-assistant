## 阶段1：数据模型 ✅

**文件**：`src/models/schemas.py` + `src/models/__init__.py`

| 部分 | 类/函数 | 类型 | 用途 |
|------|---------|------|------|
| 一 | `MeetingStatus` | `str, Enum` | 会议生命周期：created → recording → transcribing → processing → completed/failed |
| 一 | `Priority` | `str, Enum` | 任务优先级：low / medium / high / urgent |
| 一 | `SentimentType` | `str, Enum` | 情绪类型：positive / neutral / negative |
| 二 | `TranscriptSegment` | `BaseModel` | 单个语音片段（speaker, text, start, end, confidence） |
| 二 | `TranscriptResult` | `BaseModel` | 完整转写结果（segments 列表 + full_text） |
| 三 | `TopicSummary` | `BaseModel` | 单个议题摘要（讨论要点 + 结论） |
| 三 | `MeetingSummary` | `BaseModel` | 结构化会议纪要（议题列表 + 决策 + 下一步） |
| 四 | `ActionItem` | `BaseModel` | 单个待办（负责人 + 任务 + 截止日 + 优先级） |
| 四 | `ActionResult` | `BaseModel` | 待办处理结果（含 Jira/飞书同步状态） |
| 五 | `SpeakerStats` | `BaseModel` | 说话人统计（时长、占比、次数）— 规则引擎 |
| 五 | `MeetingInsight` | `BaseModel` | 会议洞察（情绪 + 效率评分 + 关键词 + 建议） |
| 六 | `FollowUpResult` | `BaseModel` | 跟进结果（发送状态 + 已创建 Issue/任务数） |
| 七 | `MeetingState` | `TypedDict` | LangGraph 共享状态（所有 Agent 读写此 dict） |
| 八 | `create_initial_state()` | 工厂函数 | 创建 Pipeline 初始状态 |

**验证**：`python demos/run_phase1.py`

## 阶段2：基础设施层 ⬜

**文件**：

| 文件 | 说明 |
|------|------|
| `src/integrations/__init__.py` | 导出三个客户端 |
| `src/integrations/minimax_client.py` | LLM 客户端：异步 HTTP + 指数退避重试 + JSON 降级解析 |
| `src/integrations/jira_client.py` | Jira 客户端：懒加载 + is_enabled 开关 + meeting-auto 标签 + 优先级映射 |
| `src/integrations/feishu_client.py` | 飞书客户端：Token 缓存（提前5分钟刷新）+ Webhook 卡片 + 任务创建 |

**核心知识点**：

| 模式 | 说明 |
|------|------|
| `is_enabled` 开关 | 外部服务未配置时自动跳过，不报错 |
| 懒加载 `_get_client()` | 只在首次调用 API 时才创建连接 |
| Token 缓存 | 飞书 token 缓存 1小时55分钟，留5分钟缓冲防 401 |
| `@retry` 指数退避 | 最多重试3次，间隔 1s → 2s → 4s |
| JSON 降级解析 | LLM 输出被 markdown 包裹时，截取 `{...}` 部分 |
| 上下文管理器 | `__aenter__` / `__aexit__` 确保 httpx 连接关闭 |

**验证**：`python demos/run_phase2.py`

## 阶段3：最小Pipeline ⬜

**文件**：

| 文件 | 说明 |
|------|------|
| `src/agents/__init__.py` | 导出 Agent |
| `src/agents/transcription_agent.py` | 转写Agent：WhisperX + pyannote 说话人识别 + demo 降级 |
| `src/agents/summary_agent.py` | 摘要Agent：Few-shot Prompt + JSON Schema + 规则降级 |
| `src/graph/meeting_graph.py` | 最小 Graph：START → Transcription → Summary → END |

**核心知识点**：

| 概念 | 说明 |
|------|------|
| 懒加载模型 | `_lazy_init()` 避免启动时加载大模型 |
| Demo 降级 | 无音频或模型未安装时，使用预设中文对话数据 |
| Few-shot Prompt | System Prompt 定角色 + User Prompt 定格式 + JSON Schema 约束 |
| 规则降级 | LLM 失败时，从文本提取说话人列表作为兜底 |
| StateGraph | `add_node` 注册节点，`add_edge` 定义流转 |
| `ainvoke()` | 异步执行 Graph，传入初始 state dict |

**Pipeline 图**：
```
START → Transcription → Summary → END
```

**验证**：`python demos/run_phase3.py`

## 阶段4：完整5-Agent ⬜

**文件**：

| 文件 | 说明 |
|------|------|
| `src/agents/action_agent.py` | 待办Agent：LLM 提取 + 同步 Jira/飞书 |
| `src/agents/insight_agent.py` | 洞察Agent：规则统计发言 + LLM 分析情绪/效率 |
| `src/agents/followup_agent.py` | 跟进Agent：Fan-in 汇聚 + Markdown 报告 + 飞书推送 |
| 更新 `src/graph/meeting_graph.py` | 添加并行节点和 Fan-out/Fan-in 边 |

**核心知识点**：

| 概念 | 说明 |
|------|------|
| Fan-out | `transcription → summary/action/insight` 三条边，LangGraph 自动并行执行 |
| Fan-in | `summary/action/insight → followup` 三条边汇聚，等所有分支完成 |
| 并行安全 | 三个 Agent 写入不同 state key：`summary`、`actions`、`insights` |
| 规则+LLM混合 | 发言统计用规则（确定快），情绪/关键词用LLM（灵活） |
| 效率评分 | 0.4×LLM评分 + 0.3×发言均衡度(基尼系数) + 0.3×时间利用率 |
| 幂等性 | Jira Issue 加 `meeting-auto` 标签，防止重复创建 |

**Pipeline 图**：
```
                    ┌─→ Summary ─────┐
START → Transcription ──→ Action ──────→ FollowUp → END
                    └─→ Insight ─────┘
```

**验证**：`python demos/run_phase4.py`
